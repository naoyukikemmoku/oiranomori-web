### 横断的な発見

**1. サイト名4種混在**
- `<title>` / フッター：里山パークおいらの森
- 構造化データ alternateName：フォレストパークおいらの森
- ヘッダーロゴ：おいらの森
- 英語表記：Satoyama Park Oiranomori
- 楽天・Instagram：フォレストパークおいらの森

**2. "強い表現"の温度差は「対象に向けるか／自分に向けるか」**
- deep-guide「期待値を調整して」← 読み手に向けてる → 遠ざける
- residents「変態的に好きだ」← 自分に向けてる → 引き寄せる
- 強さの問題ではなく、向ける先の問題。**トーン原則の核**。

**3.「100年の森」の主語問題**
- about / oiras / footer / index Hero すべてに登場
- オーナーのミッションか、キャンパーが体験するものか、書き手で揺れる

**4.「ない」フレームの散在**
- 「シャワーはありません」「コンビニも〜ない」「副産物ではない」（←最後だけポジティブ反転で機能）
- 原則化：**「ない」を使うときは必ず「代わりにある」を併記**

## 作業順序

### Step 1: サイト名統一を決める

**目的**：後続作業の基準点を確定する。SEO・OG・構造化データ・ヘッダー/フッター全ページに影響するので最初に固める。

**やること**：
- メイン名（１つ）を決める
- 英語表記を決める
- alternateName を維持するかを決める
- 影響箇所リスト：title / JSON-LD name / footer / ヘッダーロゴ / OG画像内テキスト / 楽天・Instagram の bio との整合

**成果物**：サイト名統一方針メモ（短文・選定理由付き）

**完了条件**：メイン名・サブ名（あれば）・英語名の3つが確定

---

### Step 2: トーン原則を1枚にまとめる

**目的**：修正・新規執筆の判断基準を作る。書き手のブレを防ぐ。原則ができれば後続作業はパターン作業になる。

**やること**：

**（A）良い素材から原則を抽出**
- residents「変態的に好きだ」「俺の山なんだよ」
- 「キャンプといえば焚火 / 焚き火をするためにキャンプする方も」
- 「キャンプを存分に楽しんでもらいたい。だからアーリーチェックインは無料にしています」
- deep「この雰囲気は、いかないとわからない。」
- firewood「100年後の森を、創る。」「命の欠片」

**（B）きつさパターンの言語化**
- オーナー目線 → キャンパー目線
- 強表現は対象ではなく自分に向ける
- 「ない」を使うときは必ず「代わりにある」を併記
- 禁止は理由とセットで前向きに
- 「向きません」「お断り」フレームの代替表現

**（C）主語ルール**
- 誰の物語として書くか
- 「100年の森」の使い方ルール

**成果物**：`tone-guide.md`（1枚）

**完了条件**：「OK / NG」の判断ができるレベルの原則が並ぶこと

### Step 4: deep-guide / light-guide / access の角取り

**目的**：強表現の対象問題を解消。読み手を遠ざけてる箇所を整える。

**やること**：
- 各ページごとに仕分け
  - 残す表現（必要な強さ）
  - 角を取る表現（言い換え）
  - 削る表現
- 修正案を出す

**ページ別フォーカス**：
- **deep-guide**：ディープの世界観上の「覚悟」表現は残す。「期待値を調整して」「不便の許容」など読み手に向けた断定表現を整える。
- **light-guide**：「厳守」「禁止」「サイレント」「即警察通報」の連打を、理由併記＋前向きフレームに。
- **access**：必要な強さ（爆音車両不可など安全・近隣配慮事項）は残す。「ご遠慮ください」3連発を整える。

**成果物**：各ページの差分案

**完了条件**：ナオが採用案決定

### Step 5: about / oiras の温度合わせ
