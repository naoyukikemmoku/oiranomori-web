## 1. 共通テーマ（A×B 両系統に跨る問題＝まとめて潰すべき4点）

A系統とB系統で別々に挙がっていたが、**根が同じ／同じ施策で両面解決できる**もの。ここを束で叩くのが効率的。

### 共通① お知らせ／news 運用の停滞 — [共通]
- **[A]** お知らせ一覧が2件・最新 2026-04-21 で約2か月停滞。しかもトップがこの停滞2件を前面表示 → 「営業してる？」の活発さシグナル毀損。
- **[B]** newsスタブ `/news/wkyheflxbc/`（「リニューアル準備中」desc27字）が**中身ほぼ空のままindex＋sitemap収録** → 低品質判定リスク。
- **根は同じ**：お知らせ運用が止まっていることが、UX（鮮度）と技術SEO（薄ページのindex）の両面に出ている。
- **同時解決策**：① お知らせを定期更新する or トップの「お知らせ」枠を **forest-daily（今日のおいらの森）最新**に差し替える、② スタブ記事を下書き化／410／条件付noindex。
  - 補足：生きた更新エンジンは forest-daily 側。お知らせを無理に埋めるより、鮮度のある forest-daily を前面に出す方が筋が良い。

### 共通② 画像アセットの整備不全 — [共通]
- **[A]** firewood / rental / shop / map に**写真そのものが不足**。薪・レンタル品は予約判断の核で、テキストのみでは訴求が立たない。
- **[B]** 既にある画像が活きていない：`dist` に AVIF×12・WebP×17 を**生成済みなのに `<picture>` 使用は全サイトで1ページのみ**。トップhero/bonfireはCSS背景JPEG・preload無で **LCP悪化リスク**。
- **一気通貫の解**：撮る（A）→ AVIF/WebP で配線し hero は preload/`fetchpriority`（B）。撮影と最適化を分断せず、追加するなら最初から最適化形式で載せる。

### 共通③ h1／見出しの扱い — [共通]
- **[B]** h1は**全38ページで個数が単一**（厳守）＝構造として健全。
- **[A]** ただし `/map/` だけ **h1の中身が `<title>` 文字列丸ごと**（「場内・周辺マップ ｜ 里山パークおいらの森」）。他ページのh1はクリーン（「アクセス」等）。
- **整理**：「h1の**個数**は全ページOK・`/map/` だけ**中身**がNG」。1ページの局所修正で済む。

### 共通④ 直火の構造タグ遵守 — [共通・ポジティブ]
- **[A][B] ともに、title / meta description / JSON-LD への「直火」混入はゼロ**（B系統は全38ページ実測、A系統は deep/deep-guide/faq/media の構造タグで確認）。
- プロジェクトの絶対ライン（spec §11.1・CLAUDE.md）を遵守。本文への直火残置は方針通りで問題なし。**ここは崩さず維持**。

## 3. A系統詳細（コンテンツ・UX・訴求・トーン）

### 3-1. 全体総評
トーン準拠度が高く、ここが本サイトの強み。residents・deep・index の情緒コピーは tone_guide §10 素材に忠実で、思想（100年の森）の押し出しも機能ページに侵食しておらず、「作り手の思想が前面に出すぎて予約を妨げる」問題はほぼ無い。一方で、**予約転換を直接動かす「料金・写真・鮮度・数値整合」がトーンの完成度に追いついていない**。コピーは芯があるのに、いざ予約しようとすると「料金が即わからない」「写真がない」「最新情報が2か月前」で熱が冷める。足回りの修正は大半が低工数。

### 3-2. 全体評価（要点）
- **予約導線**：CTA一貫性は良好（Header／下固定ReserveCTA／本文が planlist へ一貫）。弱点は「料金」の段が薄くHP内で総額が固まらないこと。
- **プラン別ディープリンクのURL形式が2系統混在**：トップ＝`/search/?plans=4&show=plan`（spec §9準拠）、light-guide＝`/plan/5/` 形式。spec §9 の search 形式へ寄せる（両方の挙動確認のうえ）。
- **キャンセルポリシーのリンク不整合**（§2-3）。
- **情報網羅性**：設備/アクセス/予約方法/禁止事項/サイト選び早見は充実。不足は ①HP内料金表 ②写真（firewood/rental/shop/map）③FAQの決済Q&A。
- **空き状況ウィジェット**（トップ AVAILABILITY／deep Silence）は転換にプラスの良い仕掛け（JS描画のため実挙動は本評価で未検証）。

## 4. B系統詳細（技術SEO・構造）

### 4-1. 全体総評
クロール土台はかなり堅い。sitemap / robots / canonical / JSON-LD主要タイプ（LocalBusiness+Campground / FAQPage / Article / BreadcrumbList）はテンプレート実装で全ページ自動付与、h1単一性は全38ページ厳守、`<img>` alt は 166/166 完備、内部リンクはフッター起点で孤立ページゼロ、直火の構造タグ混入ゼロ。**伸びしろは「クロールされるか」ではなく「クリックされるか・速いか」に集中**しており、その大半がテンプレート1〜2ファイルの修正で全ページ一括改善できる高ROI領域。

### 4-2. 全体評価（要点）
- **共通欠落（全38ページ・`BaseLayout.astro` で直る）**：✕ `og:url` 欠落／✕ `og:type=website` 固定／✕ description 長すぎ／⚠ canonical がクエリを乗せ得る。※`twitter:title`・`twitter:description` 欠落は実害なし（X は `og:` にフォールバック）＝要修正から除外。
- **LCP**：✕ トップhero/bonfire がCSS背景JPEG・preload無（`index.astro:120`,`221`）。
- **画像**：⚠ AVIF/WebP 死蔵（`<picture>` 1ページのみ）。CLS は寸法明示で良好。
- **P**：sitemap/robots、noindex二重防御（checkout/thanks）、完全SSG、直火点検。
- **⚠ 軽微**：sitemap に `lastmod` 無し／一覧ページに CollectionPage 等が無い（任意）。

## 6. 評価の限界・保留事項

- **[A]** `/firewood/`・`/about/`・`/news個別` はサイトのbot検知でライブ取得できず。構造面はB系統が補完済み。**firewoodの本文コピー品質・写真有無のみ未確認＝保留**。bot検知が緩んだら再取得して差分を埋める。
- **[B]** 実 LCP/INP/CLS の**実測値は未計測**。CLSは寸法明示で良好と推測、LCPは背景JPEG・preload無から悪化リスクありと判断。実数は GA4/PSI から。
- **[B]** forest-daily の og:image 外部URL（`oiranomori.com/app/sns/...php`）の可用性・寸法は未検証（外部サーバ）。
- **[A]** 空き状況ウィジェット（トップ／deep）と `/now/`・`/map/` の動的データ表示は JS描画のため実挙動を未検証。
- **判断保留（ナオ決裁）**：① newsスタブの index/410／② HP内プラン料金の表示要否（設計判断）。
