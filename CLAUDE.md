## 2. プロジェクト基本方針

### サイト基本情報
- 正式名称:里山パークおいらの森(Satoyama Park Oira no Mori)
- 旧称:フォレストパーク おいらの森(旧表記の残存に注意。発見時はチャットへ投げ返す)
- ドメイン:oiranomori.jp

### 現フェーズ
リニューアル構築は完了。DNS切替済み。現在は**運用・コンテンツ拡充・SEOチューニング**フェーズ。

### デザインコンセプト
「NOT A HOTEL」の没入感と「ゲストハウスLeu」の機能性を融合。森の中にいるような没入感を画面上で再現する。

### 禁忌事項(絶対に使わない)
- 白いカードUI
- 安易な枠線・影
- ボタンやアラートなど汎用コンポーネントを感じさせる要素
- 「キャンプ場サイトっぽい」テンプレート感

### 視覚表現の方向性
- 地面にフラットに置かれたテキスト
- 情緒的なタイポグラフィ(Noto Serif JP)
- 余白を恐れない大胆なレイアウト
- ダーク・アースカラーベース

### フィールド差別化
ライトフォレストとディープフォレストを「別サイトのように」書き分ける。

| フィールド | キャラクター |
|---|---|
| ライトフォレスト | 安心・快適・ファミキャン・ソロ初心者歓迎 |
| ディープフォレスト | 18禁・静寂・焚き火・ブッシュクラフト |

### 直火の扱い

直火は提供継続（Deep＝直火可能エリアで全面可／Light＝指定区画のみ）。終了・縮小ではない。
ただし**差別化軸としては使わない**。以下を厳守する。

- **SEO構造タグ（`<title>` / meta description / JSON-LD）に「直火」を入れない**（絶対ライン）。検索エンジンに「直火キャンプ場」として学習させない。
- 新規コンテンツで直火を推さない・前面に押し出さない・強調しない。
- 本文の既存直火記述は残してよい（淡々とした事実として。「焚き火へ置換」方針は撤回）。
- FAQ標準回答：「直火できますか？→直火エリアで可能です」。
- 詳細は **`docs/spec-2026-05.md` §11.1** を正本とする。

## 4. 実装フロー

### 基本動作
チャットから降ってくる指示は**ナオとの間で合意済み**。Codeは確認や相談を挟まず、忠実に実装する。「先走り厳禁」「案→合意→実装」のような構築期のルールは適用しない。

### コミット・プッシュ運用
- 実装単位ごとに意味のあるコミットメッセージで分割
- コミット前にリンター/フォーマッター(Biome)を通す
- プッシュは原則自走

### フィードバックループ(違和感検知時)

実装中に以下のいずれかを検知した場合、実装を**一時停止**し、`【チャット確認事項】` を冒頭に付けた出力を返す。

- 指示が既存コード・デザイン方針と矛盾する
- 禁忌事項に抵触する可能性がある
- 指示の解釈が複数あり、どれを採るかで結果が大きく変わる
- 必要な情報(画像、文言、URL等)が不足している

出力には以下を含める:
1. **検知した問題**:何が引っかかったか
2. **想定される選択肢**:A案/B案など複数提示
3. **推奨案と理由**:Code側の判断とその根拠

ナオがチャットへコピペして相談しやすい形で出力する。

## 5. SEO実装の自走原則

コンテンツ実装と同時に、以下のSEO技術要素を**自走で**整える。チャットへの確認は不要。

### 自走範囲

**メタ情報**
- `<title>`:ページごとに固有、32文字前後を目安
- `meta description`:120文字前後、ページ内容を要約
- `canonical`:全ページに付与、自己参照を基本
- `OGP / Twitter Card`:og:title / og:description / og:image / og:type / twitter:card 一式

**構造化データ(JSON-LD)**
- トップ:`["LocalBusiness", "Campground"]`(複数タイプを配列で指定)
- お知らせ・記事系:`Article` または `NewsArticle`
- FAQ要素のあるページ:`FAQPage`
- パンくず:`BreadcrumbList`(BaseLayoutで自動生成)
- 既存ページのJSON-LD実装状況はソース(`src/`配下)を正として確認する

**サイトマップ・クロール制御**
- `sitemap.xml`:自動生成・自動更新
- `robots.txt`:本番公開ページは全許可、不要なクロールはDisallow
- 404・410の適切な設定

**画像最適化**
- フォーマット:WebPまたはAVIFを基本、フォールバックはJPEG
- `alt`:装飾画像は空 `alt=""`、内容画像は意味を記述
- `loading="lazy"` をファーストビュー以外に付与
- `width` / `height` 明示でCLS抑止

**内部リンク**
- 関連ページへの導線を本文・フッターから自然に張る
- アンカーテキストはキーワード詰め込みでなく自然な日本語
- 孤立ページを作らない

### 対象外(Code側で判断しない)
- キーワード選定
- コンテンツ企画・タイトル設計
- 競合分析・ターゲット設計

これらはチャット側で決定し、確定情報として降ってくる。

## 6. パフォーマンス・アクセシビリティの自走原則

### Core Web Vitals
- **LCP**:2.5秒以内を目標。ヒーロー画像はpreload、適切なサイズ配信
- **CLS**:0.1以下。画像・iframe・広告枠は寸法明示
- **INP**:200ms以下。重いJSは遅延・分割

### 画像・アセット
- レスポンシブ画像(`srcset` / `sizes`)
- 遅延読み込みの徹底
- フォントは `font-display: swap`、必要なウェイトのみ読み込み

### アクセシビリティ最低ライン
- セマンティックHTML(`<header>` `<nav>` `<main>` `<article>` `<footer>` の適切使用)
- 見出し階層を飛ばさない(h1→h2→h3)
- フォーカス可視化を消さない
- カラーコントラスト:本文4.5:1以上
- インタラクティブ要素には適切な `aria-*` を付与
- キーボード操作で全機能が完結する

## 10. ディレクトリ・ファイル作成の禁忌

### 必須プロセス(新規作成前)
1. 新規ディレクトリ・ファイルを作成する前に、必ず既存構造を `ls` または `find` で確認する
2. 似た用途の既存ディレクトリがある場合、**新規作成せず既存を使う**
3. 命名揺れ(doc/docs、img/images、util/utils、source/src等)は自動的に避ける
4. 判断に迷ったら作らずユーザーに確認する

### 既存ディレクトリ(変更・分裂禁止)
- ドキュメント類:`docs/`(`doc/` を作らない)
- ソース:`src/`
- 公開静的ファイル:`public/`
- 廃棄ドキュメント:`docs/archive/`(旧版・構築期遺物の物理隔離先)
- 構築期参照(旧 `_references/`):`docs/archive/_references/` に隔離済み

### 違反時のリカバリ
- 誤って作成したディレクトリ・ファイルを検出した場合、内容を本来の場所に移し、誤生成側を削除する
- 移動後は `git status` で結果報告する

## 11. 実装済みページ・機能の記録

### forest-daily（今日のおいらの森）
- **実装日**: 2026-06-04
- **URL**: `/forest-daily/`（一覧）、`/forest-daily/[id]/`（個別）
- **クライアント**: `src/lib/forestDaily.ts`
- **microCMSエンドポイント**: `forest-daily`（リスト形式）
- **コンテンツID規則**: daily=`YYYY-MM-DD`、seasonal/special=`YYYY-MM-DD-<type>`
- **書き込み主体**: OirasApp（VPS）がマネジメントAPI PUTで自動投稿
- **フィールド**: title / body(textArea) / postDate / postType(string[]) / imageUrl / videoUrl / tempC / maxTempC / minTempC / humidity / condition
- **body**: プレーンテキスト。`bodyToParagraphs()` で改行→`<p>`変換はHP側が担う
- **postType判定**: `.includes()` ベース（配列複数選択対応）
- **imageUrl欠落時**: videoUrl YouTubeサムネイル → デフォルトhero の順でフォールバック
- **気象表示**: 各フィールド個別ガードで縮退対応。`flex flex-wrap` で部分欠落時もレイアウト崩れなし
- **待機事項**: OirasApp HpAdapter の疎通テスト（実PUT→ページ自動生成）確認待ち
