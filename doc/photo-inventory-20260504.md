## 1. /firewood/

### 現状画像

- ファイル: `/images/bonfire.jpg`
- 配置: Hero セクション（CSS background-image）
- レイアウト: 全幅背景画像オーバーレイ（`lp-hero` クラス）

**これ以外の画像は一切なし。**

### 現状のページ構成と画像の関係

| セクション | id | 背景 | 画像有無 |
|---|---|---|---|
| Hero | — | `bonfire.jpg`（焚き火炎のアップ） | ○（ただし薪とは無関係） |
| 販売している薪の種類 | `#sales` | ライトグレー | ✗ |
| 薪の種類（カード型） | `#species` | ダークグレー（zinc-800） | ✗ |
| セルフ量り売りの手順 | `#steps` | ライトグレー | ✗ |
| なぜ・・・よく燃えるのか | `#quality` | ダークグリーン（forest-700） | ✗ |
| 薪ができるまで | `#process` | ダークグリーン（forest-600） | ✗ |
| FAQ | — | ライトグレー | ✗ |
| Related | — | ダーク | ✗ |

### 不足・改善箇所

1. **Hero画像の不一致**: `bonfire.jpg` は焚き火の炎のアップで、「薪の質・種類」を伝えるページのヒーローとしてはミスマッチ。薪棚・玉切り・積み上げた薪の写真の方がページ目的と合致する。
2. **`#species`（薪の種類カード）**: 8種の薪カードに写真がなく、テキスト情報のみ。各樹種の実物（断面・樹皮）が見えると購買意欲が上がる。
3. **`#process`（薪ができるまで）**: 伐採・玉切り・薪割り・乾燥の4ステップが写真なしで説明されている。林業プロとしての「見せどころ」セクションだが視覚的に弱い。
4. **`#quality`（なぜよく燃えるのか）**: 含水率15%以下・自家生産という強みをテキストのみで語っている。薪棚写真や水分計の写真があると説得力が大きく増す。

### 推奨アクション

- 追加枚数: **3〜5枚**
- 優先順位:
  1. Hero画像の差し替え（薪棚または積み上げた薪）
  2. `#process` セクションへのステップ写真（各1枚 × 4ステップ、または代表1〜2枚）
  3. `#quality` セクションへの薪棚・乾燥中の写真
- レイアウト候補:
  - `#process`: 4カラムグリッドの各ステップ上部に小さめの正方形写真（既存 LP プランカードのパターン応用）
  - `#quality`: 2カラム（テキスト左・画像右）— 既存 LP EARLY CHECK-IN の踏襲

## 2. /shop/

### 現状画像

**画像ゼロ。**

`PageLayout`（標準ページ）を使用。ヒーローなし、ページ内の全セクションがテキスト・テーブルのみ。

### 現状のページ構成と画像の関係

| セクション | 画像有無 | 補足 |
|---|---|---|
| ページヘッダー | ✗ | テキストのみ |
| 取り扱い商品（6カテゴリ） | ✗ | カテゴリ名＋商品テキストのみ |
| セルフ精算の流れ | ✗ | SCAN/SELECT/PICK UP のステップ |
| 量り売りテーブル | ✗ | 生ビール・ハイボール・オイル・灯油 |

### 不足・改善箇所

1. **売店内観の不在**: 「キャンプに必要なひと通りを揃えています」という文言に対して、何も視覚情報がない。売店の外観・内観写真が1枚あるだけでページの信頼度が大きく変わる。
2. **生ビールサーバーの写真**: LP の `#draft-beer` セクションから `/shop/` へ誘導している動線があるが、売店ページには生ビールの写真が一切ない。LP からの期待値とページの落差が大きい。
3. **産直食材**: とちおとめ・アユ・原木舞茸等、想像力を刺激する商品が並んでいるが写真なし。

### 推奨アクション

- 追加枚数: **2〜3枚**
- 優先順位:
  1. ページトップの売店外観または内観写真（ヒーロー相当。現在の `PageLayout` にはヒーローがないため、ページヘッダー直下への2カラム挿入が自然）
  2. 生ビールサーバーの写真（量り売りセクション横に配置。現在 `lp-draft-beer.jpg` が `public/images/lp/` に存在しており、LP との共用も可）
  3. 産直食材の写真（ドリンク or 食材カテゴリ横）
- レイアウト候補:
  - ヘッダー直下: 全幅 or 2カラム（画像左・ページ概要テキスト右）
  - 各カテゴリ内: `border-t` セクション内でテキスト右横に小画像（`w-24〜32` 程度のサムネイル）

## 3. /rental/

### 現状画像

**画像ゼロ。**

`PageLayout`（標準ページ）を使用。ヒーローなし、全セクションがテーブルのみ。

### 現状のページ構成と画像の関係

| セクション | 画像有無 | 補足 |
|---|---|---|
| ページヘッダー | ✗ | テキストのみ |
| お申し込み方法 | ✗ | dl グリッド |
| レンタルアイテム（8カテゴリ 40+件） | ✗ | 全カテゴリがテーブル形式 |
| テント設営・撤収サービス | ✗ | テーブル |
| 返却ルール | ✗ | dl グリッド |

### 不足・改善箇所

1. **設営済みテントの写真の不在**: レンタルの主力商品がテントだが、実際に設営された状態の写真が一切ない。「森の中で設営されたテント」の写真は予約意欲に直結する。
2. **「手ぶらで来れる」訴求の視覚化不足**: ページのコピー「手ぶらでも、足りないものだけでも。」という訴求がビジュアルで補強されていない。
3. **カテゴリのスキャナビリティ**: 8カテゴリ・40件超のテーブルが続くため、カテゴリ冒頭に代表品の写真があるとスキャンしやすくなる。

### 推奨アクション

- 追加枚数: **1〜2枚**（全カテゴリに写真を入れる必要はない）
- 優先順位:
  1. ページトップ（ヘッダー直下）に設営済みテント写真 1枚 — 「森の中でテントが張られた様子」。`_candidates/IMG_1689.JPEG` が候補として適合する可能性あり
  2. 代表カテゴリ（テント・タープ or 焚き火）の冒頭に小さなイメージ写真を1枚
- レイアウト候補:
  - ヘッダー直下: 2カラム（画像左・申し込み方法右）— shop と同じパターン
  - カテゴリ内: カテゴリ見出し横の小さな写真（h-24 程度、object-cover）

## 4. /map/

### 現状画像

- `/images/light-field-googlemap.jpg` — Google Maps の GroundOverlay として使用（場内マップの地図上に重ねる航空写真風画像）
- その他の `<img>` タグ: なし

### 現状のページ構成と画像の関係

`/map/` は **フルスクリーンインタラクティブマップアプリ** として設計されており、`height: 100dvh` で画面全体を占有する。従来の「テキスト+画像のセクション」構造を持たない。

| 要素 | 詳細 |
|---|---|
| タブバー | 場内マップ / 周辺案内（固定高さ） |
| マップエリア | Google Maps 全幅（flex: 1） |
| サイドパネル | チェックボックスフィルター（PC） / ドロワー（Mobile） |
| 周辺案内リスト | スポット一覧パネル |

### 不足・改善箇所

このページに「場内雰囲気が伝わる写真」を追加するためには、**現在の構造を変更する必要がある**。現状は 100dvh のマップ画面であり、写真を追加できる余白が存在しない。

選択肢:
- A案: マップ起動前の「ランディング状態」として簡単なヒーロー帯（写真＋説明文）を設け、スクロールまたはタブ選択でマップに移行する構造
- B案: タブバーの上に固定の細いバナー写真帯（height: 120〜160px）を追加する
- C案: 現状を変えず、代わりにマップの `?view=light` 初期表示を改善する（場内写真を InfoWindow に追加する等）

**写真追加のみでは対応不可**。構造変更の要否についてはナオ・チャット側で判断が必要。

### 推奨アクション

- `_candidates/IMG_1689.JPEG`（昼のオートサイト全景）は場内雰囲気を伝えるのに適している
- 構造変更が決定した場合に使用を検討
- 現時点では spec 9.3 の「写真追加」の文脈では保留とし、構造改善タスクとセットで扱うことを推奨

## 6. 全体の所感

### 深刻度の順位

1. **`/shop/`（最優先）**: 画像ゼロ + LP から生ビール目当てで訪れるユーザーがいる動線があるにもかかわらず、ビジュアルが何もない。売店内観写真1枚で改善幅が最も大きい。
2. **`/rental/`（優先）**: 画像ゼロ + 40件超の商品リスト。設営済みテント写真1枚でページの説得力が大幅に変わる。追加コストが最小。
3. **`/firewood/`（中優先）**: Hero は bonfire.jpg の流用で目的ずれ。ただし LP が先にある導線のため、LP の焚き火セクションで補完されている面もある。「薪ができるまで」セクションへの写真追加が最も効果的。
4. **`/map/`（要検討）**: 写真追加自体は構造変更を伴うため、独立したタスクとして扱うべき。候補画像は確保済み。

### 候補画像の評価

現在の5枚の候補は全てキャンプシーン（テント・夜景・冬の林）であり、薪・売店・レンタル品の実物写真は含まれていない。**spec 9.3 の真の要求（薪の質・売店内観・レンタル品の実物）を満たす写真はナオ側での新規撮影が必要**。候補画像は補完的用途（雰囲気・設営例）に限定して活用できる。

### 共通課題

- `PageLayout` を使う3ページ（shop/rental/map）にはヒーロー/視覚的入口がなく、テキストから始まる。ヒーロー帯を設けるかどうかはデザイン方針の判断が入るため、各ページのアクション実装前にチャット側での確認を推奨。
- firewood は `LpLayout` を使っており、ヒーローが存在する。ここのみ差し替えで対応可能。
